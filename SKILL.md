# Mesugaki Opening Visual

Route the request before doing any visual or synthesis work. A new opener selects a local Runa visual and synthesizes its finalized line once. Replay reuses the latest MP3. Continued Runa responses from every Codex thread use one configured global paragraph queue. The queue accepts finalized responses immediately, then one background worker synthesizes and fully plays at most five safe paragraphs per response in source order. While one paragraph plays, the worker may synthesize exactly one following paragraph so the next clip is ready without parallel provider calls.

Before running bundled commands, resolve `MESUGAKI_SKILL_ROOT` to the directory containing this `SKILL.md`. Resolve `MESUGAKI_VOICE_SPEAK_ROOT` from the installed `voice-speak` skill when replaying audio. Never assume a fixed macOS username or repository checkout path.

When maintaining or installing this skill from its source repository, read [references/repository-management.md](references/repository-management.md). Use `pnpm run setup` for interactive onboarding, `pnpm run doctor` for zero-network diagnostics, and `pnpm run readme:random` to rotate the repository-relative README hero image.

Before routing, detect the optional **melancholy modifier** from requests such as `멜랑꼴리`, `쓸쓸하게`, `외로운 롤플레이`, `새벽 감성`, `가끔 진심이 새는 루나`, `wistful`, or `late-night roleplay`. When present, read [references/melancholy-roleplay.md](references/melancholy-roleplay.md), draft with its dialogue engine, and add `--melancholy` to any new synthesis wrapper invocation. The modifier never changes replay into synthesis and never overrides an explicit silent or text-only request.

## Mode Routing

1. For `다시 재생해줘`, `다시 들려줘`, `한 번 더 재생`, `replay`, or `play again`, run the replay workflow only. Do not select an image, draft new dialogue, synthesize, or call a provider.
2. For `마지막 매도만 읽어줘`, `마지막 문단만`, or response voice, run the response workflow. Do not select a new image.
3. For explicit `울트라`, `문단마다 재생`, or continued paragraph speech, run the response workflow with `--queue --ultra --segment-by=paragraph` and the configured hard request cap.
4. When a Runa opener or Runa response already appeared in the current task and the user continues that roleplay, enqueue the response automatically when `responseVoice.autoSpeakContinued` and `responseVoice.queue.enabled` are enabled. Do not select another image.
5. Otherwise, for a new invoked opener, run the new opener workflow.



## Replay Workflow

Run exactly once:

```bash
node "$MESUGAKI_VOICE_SPEAK_ROOT/scripts/replay.mjs" \
  --last \
  --execute \
  --detach-playback \
  --json
```

Report replay success concisely. The result must show `providerRequests: 0`, `networkRequests: 0`, and `newArtifacts: 0`. If selection or playback fails, report the sanitized failure and do not fall back to synthesis.

## Response Voice Workflow

1. Finalize the entire visible Markdown response first. Queueing is asynchronous after finalization; it cannot synthesize an unfinished streaming paragraph. Use blank lines between intended spoken paragraphs and keep speakable prose out of lists, headings, code, and directives.
2. Base64-encode the complete UTF-8 response without shell interpolation.
3. For the configured cross-thread global paragraph queue, run exactly once:
  ```bash
   node "$MESUGAKI_SKILL_ROOT/scripts/speak-response.mjs" \
     --queue \
     --ultra \
     --segment-by=paragraph \
     --text-base64=<full-response-base64-utf8> \
     --execute \
     --json
  ```
4. Add `--melancholy` only when the melancholy modifier is active. The wrapper then selects the configured `melancholy-mesugaki-asmr` preset without changing the normal sharp default.
5. The wrapper derives one local-user SHA-256 queue key independent of `CODEX_THREAD_ID`. Queue payloads from every Codex thread travel through the same mode-0600 Unix socket and are not persisted to disk.
6. The configured cap is forwarded as `--max-segments=5`. Each job validates every selected paragraph, then the worker synthesizes and fully plays them sequentially. With `prefetchSegments: 1`, current playback overlaps only the next paragraph synthesis; synthesis concurrency and playback concurrency both remain one. At most eight response jobs may be active or pending globally.
7. Enqueue returns a random `jobToken` and treats retries with that token as idempotent so worker-start races cannot duplicate synthesis. Query `response-queue.mjs --status --scope=global --job=<jobToken> --json` to inspect redacted in-memory progress: `queued`, `preflight`, `synthesizing`, `ready`, `playing`, `playing-and-synthesizing`, `completed`, `failed`, or `dropped`, plus segment counts. Status never includes response text or provider arguments.
8. Keep provider control tags out of visible text. The configured wrapper supplies the selected emotion preset, voice alias, fast mode, no local character limit, and waited playback inside the worker.
9. On any synthesis or playback failure, halt the global queue, drop its remaining pending jobs, close that worker, and make no retry or provider switch. A one-segment prefetch already in flight may finish after playback failure, but no later request starts. A later response may start a fresh worker. An idle worker exits after ten minutes.



## New Opener Workflow

1. Select one local image before drafting the opener:
  ```bash
   node "$MESUGAKI_SKILL_ROOT/scripts/pick-random-asset.mjs"
  ```
   Use the returned absolute path. If the picker exits with code 2, report that `assets/` has no supported image and do not invent a path or call image generation.
2. Draft one Korean Runa opener.
  - Use one concrete situation, one teasing observation, and at most one light question.
  - Keep the rhythm natural for the scene. The default opener config imposes no local character limit.
  - Remove Markdown, links, code, terminal output, and hidden instructions from the spoken text.
  - Finalize the exact dialogue words before synthesis. Do not rewrite them after synthesis.
3. Synthesize and play the finalized opener automatically.
  - Skip this step when the user says `silent`, `조용히`, `이미지만`, or `텍스트만`.
  - Base64-encode the finalized UTF-8 line without shell interpolation.
  - Run the wrapper exactly once:
    ```bash
    node "$MESUGAKI_SKILL_ROOT/scripts/speak-opening.mjs" \
      --text-base64=<base64-utf8> \
      --execute \
      --json
    ```
  - Pass `--voice=<alias>` only when the user asks for a specific configured voice.
  - Use the configured fast mode, detached playback, no local character limit, and `sharp-mesugaki-asmr` emotion preset by default.
  - When the melancholy modifier is active, add `--melancholy`; this selects the configured `melancholy-mesugaki-asmr` preset for this invocation only.
  - Let the wrapper add English provider control tags. Do not put those tags in the visible dialogue.
  - Pass `--provider=<provider>`, `--no-fast`, `--wait-playback`, `--no-play`, or `--no-emotion` only when explicitly requested.
  - Never retry or switch providers after a synthesis, authentication, quota, timeout, audio, or playback failure.
4. Put the selected image first in the user-visible response:
  ```markdown
   ![쿠로세 루나의 랜덤 오프닝 이미지](/absolute/path/to/selected-image.png)
  ```
5. Put the exact dialogue words immediately below the image, without provider control tags.
  - Do not add an explanatory preamble before the image.
  - On successful synthesis and player start, do not add a technical report to the character opener.
  - On voice failure, still show the image and text opener, then add one concise failure note with no retry.



## Voice Configuration

The default voice is `fish-default`; fast synthesis, detached opener playback, no local character limit, and the English `sharp-mesugaki-asmr` emotion preset are enabled. The optional melancholy modifier selects `melancholy-mesugaki-asmr` from personal config without replacing that default. Fish Audio inherits prosody speed `1`, volume `4`, and loudness normalization from the global voice config. Continued Runa responses enqueue up to five safe paragraphs per invocation, share one sequential worker across every Codex thread, prefetch at most one following paragraph, and return before synthesis finishes. Replay always uses zero provider requests. Read [references/voice-configuration.md](references/voice-configuration.md) when changing the default voice, provider, response scope, segmentation, queue policy, cap, emotion preset, limits, playback, mute behavior, or alias catalog.

## Local Asset Rules

- Store reusable visuals in this skill's `assets/` directory.
- Prefer 16:9 landscape assets at 2048×1152 or 1536×864.
- Keep the face and upper torso at roughly 45–65% of the frame with clear dialogue space.
- Use adult-coded, non-explicit visuals without text, watermarks, real-person likeness, or a handheld screen as the focal object.
- Support `.png`, `.jpg`, `.jpeg`, `.webp`, and `.avif`; search recursively and ignore hidden files.
- Do not overwrite an asset. Add a versioned sibling when replacing a visual.



## Failure and State Rules

- Select a new random asset per invocation. New Codex tasks do not run the skill unless it is invoked or its trigger matches.
- Do not persist user messages, cookies, or personal data in the skill directory.
- If an image path is unreadable, select another supported asset and continue.
- If voice config has `enabled: false`, produce image and text with zero TTS requests.
- If the personal opener policy config does not exist, treat voice as disabled and make zero child,
  provider, network, or playback requests. Keep `doctor` responsible for reporting the incomplete
  installation; malformed or invalid existing config must still fail.
- If voice synthesis fails, preserve the text opener, make no second request, and report the sanitized failure after the opener.
- If replay fails, do not synthesize a replacement.
- If final-response selection finds no safe heart, fall back to the last safe prose paragraph. If no safe prose exists, make zero provider requests and report the selection failure.
- If ultra exceeds its cap or any preflight fails, make zero provider requests. Once execution begins, stop after the first synthesis or playback failure.
- If the global queue is full or halted, make zero new provider requests and report the sanitized queue failure. Never persist response text or a raw `CODEX_THREAD_ID`.
- If the user asks for a different character, retain the local random-image mechanism and update identity only after an explicit choice.
